Pregunta 1

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet

  • Sistema de computadoras o dispositivos como cables, routers, teléfonos, etc. que forma una red mundial de comunicación centralizada a través del protocolo TCP/IP y que permite intercambiar o consultar una variedad enorme de tipos de datos como vídeos, imágenes, páginas web, textos y demás.

World Wide Web

  • Es un sistema que funciona a través de internet, utiliza el sistema de Localizador Uniforme de Recursos o URL, se pueden enviar distintos tipos de datos a través del Protocolo de Transferencia de Hipertextos o HTTP, fue creado en HTML (Hypertext Markup Language).

Pregunta 2

¿Cuáles son las partes de un URL?

URL

  • URL (siglas del inglés: Uniform Resource Locator, es decir, Localizador Uniforme de Recursos).

Partes de URL

  • Protocolo de red
    • Http, Https, mailto y ftp son los principales protocolos web que encabezan una URL

  • Servicio
    • WWW, WWW2, etc.

  • Dominio
    • Comercial(.com), educativo(.edu), etc.

  • Ruta y nombre del archivo
    • Carpetas y directorios donde se ubica el recurso en el servidor.

Pregunta 3

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET

  • Este método solicita una representación de un recurso específico, sólo recupera datos o información.

HEAD

  • Este método se utiliza para obtener información sobre un determinado recurso, como GET pero sin retornar el registro.

POST

  • Sirve para entregar una entidad a un determinado recurso, frecuentemente cambiando el estado o efectos secundarios en el servidor.

PUT

  • Sirve para actualizar los datos a través de un servicio web REST.

PATCH

  • Sirve para aplicar modificaciones parciales a un recurso.

DELETE

  • Borrar un recurso específico.

Pregunta 5

Códigos 200, 400 y 500

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

  • Los códigos en 200 significan que todo se ha recibido, procesado y se ha regresado de manera satisfactoria, el 200 específicamente significa que todo se ha procesado de forma correcta.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

  • Los 400 indican errores del cliente, el error más famoso es el 404 y significa que el recurso no está disponible en el servidor, quizá estuvo disponible en algún momento pero ha sido borrado o no se ha escrito la dirección de manera correcta.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

  • Los errores 500 se refieren a un error del servidor y el cliente no tiene forma de arreglarlo, el error 500 es el segundo error más famoso y se refiere a que hubo un error en el servidor y no especifica nada.

Pregunta 6

HTML5

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

  • Los tags y atributos son desaprobados porque el estándar de HTML ya no los recomienda y sucede porque a veces son reemplazados y/o se vuelven obsoletos.

    Ejemplos

    • applet: Reemplazado por el tag HTML object.
    • dir: Reemplazado por el tag HTML ul.
    • isindex: Reemplazado por el tag HTML input.
    • menu: Reemplazado por el tag HTML ul.
